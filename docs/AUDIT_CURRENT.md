## КЛЮЧЕВЫЕ РАСХОЖДЕНИЯ С ТЗ

### 1. Технологический стек

**Проблема:** Текущая версия на Angular, ТЗ требует Vue 3.

**Варианты решения:**
1. **Переписать на Vue 3** — соответствие ТЗ, но потеря текущей работы
2. **Обновить ТЗ под Angular** — сохранение работы, изменение документации
3. **Параллельная разработка** — новый проект на Vue, миграция функционала

**Рекомендация:** Обсудить с заказчиком. Если Angular устраивает — обновить ТЗ.

### 2. UI Library

**Проблема:** Material Design вместо PrimeVue.

**Влияние:** Косметическое. Обе библиотеки качественные.

### 3. Отсутствие ключевого функционала

**Проблема:** Нет AI-диалога, матчинга профессий, интеграций.

**Статус:** Критические функции (Фазы 2-3) не реализованы.

## РЕКОМЕНДАЦИИ

### Вариант A: Продолжить на Angular

**Преимущества:**
- Сохранение текущей работы
- Angular — зрелый фреймворк
- Material Design хорошо подходит

**Требуется:**
- [ ] Обновить ТЗ (заменить Vue на Angular)
- [ ] Обновить смету (пересчитать часы)
- [ ] Добавить Angular Material вместо PrimeVue
- [ ] Реализовать функционал Фаз 2-4

### Вариант B: Переписать на Vue 3

**Преимущества:**
- Полное соответствие ТЗ
- Vue проще для небольших команд
- PrimeVue богаче компонентами

**Требуется:**
- [ ] Начать с нуля (Фаза 1)
- [ ] Перенести дизайн/логику из Angular
- [ ] Потеря времени на текущую версию

### Вариант C: Гибридный

**Описание:** Использовать текущий Angular для авторизации и платежей, Vue для основного функционала (микрофронтенды).

**Сложность:** Высокая, не рекомендуется для MVP.
